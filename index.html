What RESI is

A real product surface, not a demo wrapper.

RESI is built for people who need to direct agent work, understand what happened, and keep the output trail intact. It treats launch, observability, and artifacts as one workflow.

Launch

Start work with intent

Compose a task, pick the orchestration pattern, and run locally or against remote infrastructure from the same surface.

Observe

See execution in motion

Watch agents, messages, topology, and output as they happen instead of guessing what the system is doing.

Review

Inspect the actual artifacts

Browse files, logs, and structured outputs in-place so each run leaves behind something auditable and usable.

Core surfaces

Everything important stays visible.

The interface is organized around the moments that actually matter: starting work, tracking live progress, understanding the graph, and reviewing the resulting workspace.

Compose

Launch runs without configuration sprawl.

The prompt surface keeps the core controls close: task, pattern, target, model, effort, and workspace reuse.

Direct controlStart real work from a single input, not a maze of forms.
Pattern-awareChoose the orchestration style that matches the task.

Output

Watch the run produce actual deliverables.

Track agent roles, file activity, messages, and logs in one place while the run is happening.

Topology HUD

See the graph instead of inferring it.

The HUD exposes active nodes, dependencies, messaging edges, and run state as a live control surface.

Operational clarityFit, pause, and inspect the topology while work is still in flight.

Files

Review the workspace like a product output, not a side effect.

Every run has a workspace. RESI lets you browse it directly, preview contents, and pull the whole bundle when needed.
